Welcome to the HRDC Intranet

This intranet is designed to help HRDC employees and volunteers access timecards, forms, announcements, and other essential resources.

Group 5 Product Website

Our product is an intranet site requested by HRDC to assist employees and volunteers in accessing timecards, forms, and other necessary information.

Key Features

  • **Search Functionality** – Users can browse the HRDC Google Drive for any required forms.
  • **Admin Announcements** – HRDC admins can post updates such as timecard due dates, training sessions, and W-2 form availability.
  • **User Logins** – Employees and volunteers can log in to access personalized documents (W-2s, training materials, payroll forms).
  • **Easy Access to HRDC Forms** – Frequently used forms are pinned for quick retrieval.
  • **User-Friendly Design** – Designed for elderly users, visually impaired individuals, and non-native English speakers.

User Documentation

This intranet will be linked to the main HRDC website via a subdomain, ensuring easy access for users. The interface is designed to minimize the need for HRDC admin assistance when finding and filling out forms.

Developer Documentation

  • **Source Code**: Available on GitHub.
  • **Framework**: Built using **Next.js** for a fast, responsive interface.
  • **Code Documentation**: Follows best practices to ensure easy maintenance.

How to Obtain Source Code

The source code is accessible through the GitHub project. The final project will also be hosted online.

Planned Directory Structure

We are using a two-level directory structure to separate directories for each user type, ensuring better organization.

How to Build & Test

  • **Build Process**: The website is built using **Next.js**, with additional files for future features.
  • **Testing**: We conduct **black-box testing**, especially with non-technical users and elderly individuals.
  • **Automated Testing**: Not currently implemented but will be added as needed.

Version Releases & Bug Tracking

  • **Releasing Versions**: Each new release is published on GitHub.
  • **Bug Reporting**: Developers and users can report issues through the GitHub Issues page.

Code Documentation Best Practices

We follow strict documentation practices to ensure code clarity and maintainability:

  • Comments should **not duplicate the code**.
  • Clear comments should explain **why the code exists**, not just what it does.
  • If the comment is unclear, **the code should be rewritten**.
  • Provide links to **external sources and references** where needed.
  • Always **comment when fixing a bug** or marking incomplete implementations.
