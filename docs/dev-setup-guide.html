HRDC Intranet Local Setup Guide

Running the App Locally (New in this release)

This section explains how developers can run the HRDC Intranet application locally for development or testing purposes.

System Requirements

  • Node.js version: The application requires Node.js v18.20.7. Other versions may lead to Firebase compatibility errors.
  • Package Manager: npm is used for dependency management.

Initial Setup

  1. Install Node Version Manager (nvm) if not already installed. Instructions can be found here.
  2. Run the following commands in your terminal: 
    nvm install 18.20.7
nvm use 18.20.7
  3. Clone the project from GitHub: 
    git clone https://github.com/423S25/repo5.git
cd repo5
  4. Install project dependencies: 
    npm install

Firebase Configuration

Each developer must connect the app to their own Firebase project to use login and announcement features.

  1. Create a Firebase project at Firebase Console.
  2. Enable Authentication (email/password and optionally Google).
  3. Enable Firestore Database.
  4. Create a web app in Firebase and copy the configuration object.
  5. Create a file named .env.local in the root of the project and paste the configuration values in this format:
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_messaging_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id

Running the App

To start the development server, run:

npm run dev

This will launch the app at http://localhost:3000.

Note: If the app fails to start or login throws errors, double-check that:

  • You are using Node.js version 18.20.7
  • Your .env.local file is present and contains valid Firebase keys
  • You have access to Firestore and Firebase Authentication in your Firebase project
