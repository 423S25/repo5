Authentication

  • Firebase Authentication: Implemented email/password and Google login using Firebase Auth.
  • User Roles: Authenticated users are assigned roles based on their login email.
  • Session Handling: Logged-in users persist via Firebase and are redirected on login/logout.

Announcements Feature

  • Firestore Integration: Announcements are stored in Firebase Firestore.
  • Display: Announcements appear dynamically on the homepage for all users.
  • Admin Access: Only admin@hrdc.com (password: admin123) can create, edit, or delete announcements.
  • Editing: Admins can now edit announcements directly from the Manage Announcements page using the Edit button next to each post. (New in this release)
  • Security Rules: Firestore is secured to allow read for all, but write only for the admin.

Links Page Feature (New in this release)

  • Firestore Integration: Links are stored in Firebase Firestore.
  • Display: Links are displayed on the Links page and accessible to all users.
  • Admin Access: Only admin@hrdc.com (password: admin123) can add, edit, or delete links.
  • Link Management: Admins can manage links directly from the Manage Links page, including adding new links, editing existing ones, and deleting unwanted links.
  • Security Rules: Firestore is secured to allow read for all, but write only for the admin.

Google Calendar (New in this release)

  • Google Calendar Integration: A shared HRDC calendar was embedded using an <iframe> element in src/app/page.tsx.

    • Testing

    • User Testing Summary: We conducted structured user testing with five participants, none of whom had prior experience with our system. Users completed tasks like logging in, posting announcements, using search, and locating key resources. Their feedback led us to improve login input field visibility and underline and color important links. A one-page overview is available in the User Evaluation Summary. Full testing report can be found in the HRDC User Testing Report, along with a Supplemental Test Case Log.
    • Unit Tests Implemented: We added tests using @testing-library/react for several parts of the site: the Help Page, Search Page, Navbar, and a utility function called normalize, which cleans up text by removing extra spaces and ignoring capitalization (in Search). These tests check if pages show the right things, if buttons and inputs work, and if content changes based on the user (admin vs. regular user). All test files are in the __tests__ folder.
    • System Tests: We also created system-level tests for the Home Page and Announcements. These tests pretend to load data from Firebase to make sure the right content shows up on the page. This helps us check how the app works as a whole.
    • Manual Testing: Manual testing was performed for login/logout, posting announcements, announcement display, and role-based access (admin vs. general users).
    • Selenium IDE Test: An end-to-end test was created using Selenium IDE to simulate admin login, posting an announcement, and logout. The test file is included in the root directory as PostAnnouncementTest.side. You can download the test .side file here.
    • Human-Computer Interaction Tests: Since our webpage is intended for non-technical and elderly users, Human-Computer Interaction (HCI) principles were also considered, including accessibility, color contrast, and navigation clarity using tools such as those listed on the W3C Accessibility Testing Tools site.

    HRDC Transfer

    Option 1

    The simplest option is to keep the domain name the site is currently under. Here they can put a button on their main page that links to this site. Here the benefit is how simplistic this process is. A simpler process leads to less change of error and issues. The downfall of doing it this way is that the url and domain name will not be inline with the HRDC.

    Option 2

    Another way to do this is to transfer the site to be under HRDCs hosting on wordpress. Here are the steps to complete this.

      SiteGround
      A2 Hosting
      Namecheap

  • Set up an account and during the signup process, choose or configure your custom domain name.
  • Clone GitHub Repo or download

  • Upload to the web host
      • Use an FTP client like FileZilla or Cyberduck to connect to your hosting server.
      Log in with your hosting account credentials (usually provided in the welcome email)
      Connect using the FTP credentials (host, username, password, port).
      Upload the files from your extracted repo folder
  • Link your domain to the hosting account, ensuring that the domain’s DNS records are updated.
  • Ensure correct file permissions & settings
  • Test the site and domain
