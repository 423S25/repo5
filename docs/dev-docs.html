Project Structure

The project follows a structured directory setup to separate user and admin functionalities for better organization.

Build Process

  • Framework: Built using Next.js as the primary technology.
  • Future Development: Additional features and files will be integrated in upcoming iterations.

Authentication (New in this release)

  • Firebase Authentication: Implemented email/password and Google login using Firebase Auth.
  • User Roles: Authenticated users are assigned roles based on their login email.
  • Session Handling: Logged-in users persist via Firebase and are redirected on login/logout.

Announcements Feature (New in this release)

  • Firestore Integration: Announcements are stored in Firebase Firestore.
  • Display: Announcements appear dynamically on the homepage for all users.
  • Admin Access: Only admin@hrdc.com (password: admin123) can create announcements.
  • Security Rules: Firestore is secured to allow read for all, but write only for the admin.

Testing (Expanded since Beta Release)

  • Testing Focus: Primarily black-box testing with usability testing for non-technical and elderly users. We will also focus on HCI testing, this would include accessibility and testing users who fit our target profile (elderly and non-technical).
  • Automated Testing: We will be adding in automated testing along with black box and HCI. We will refer to this link on resources to do so https://www.w3.org/WAI/test-evaluate/tools/list/

Code Documentation Best Practices

  • Comments should provide context, not just describe the code.
  • If the comment is unclear, the code should be rewritten for better clarity.
  • Use external references where needed for complex logic.
  • Bug fixes should always be documented in comments.
