HRDC Intranet GitHub Repository

Project Structure

The project follows a structured directory setup to separate user and admin functionalities for better organization.

Build Process

  • Framework: Built using Next.js as the primary technology.
  • Future Development: Additional features and files will be integrated in upcoming iterations.

Testing

  • Testing Focus: Primarily black-box testing with usability testing for non-technical and elderly users.
  • Automated Testing: Currently not implemented but planned for future updates.

Release Process

New releases are published on the project's GitHub repository.

Bug Tracking

Issues are tracked through GitHub Issues. Developers should report, document, and resolve issues efficiently.

Submit bug reports here: GitHub Issues Page

Code Documentation Best Practices

  • Comments should provide context, not just describe the code.
  • If the comment is unclear, the code should be rewritten for better clarity.
  • Use external references where needed for complex logic.
  • Bug fixes should always be documented in comments.
