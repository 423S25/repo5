Project Structure

The project follows a structured directory setup to separate user and admin functionalities for better organization.

Build Process

  • Framework: Built using Next.js as the primary technology.
  • Future Development: Additional features and files will be integrated in upcoming iterations.

Running the App Locally (New in this release)

For full instructions on how to run the HRDC Intranet locally, including environment setup and Firebase configuration, see the Local Setup Guide.

Authentication

  • Firebase Authentication: Implemented email/password and Google login using Firebase Auth.
  • User Roles: Authenticated users are assigned roles based on their login email.
  • Session Handling: Logged-in users persist via Firebase and are redirected on login/logout.

Announcements Feature

  • Firestore Integration: Announcements are stored in Firebase Firestore.
  • Display: Announcements appear dynamically on the homepage for all users.
  • Admin Access: Only admin@hrdc.com (password: admin123) can create or delete announcements.
  • Deletion: Admins can now delete announcements directly from the Manage Announcements page using a Delete button next to each post. (New in this release)
  • Security Rules: Firestore is secured to allow read for all, but write only for the admin.
  • Google Calendar Integration: A shared HRDC calendar was embedded using an <iframe> element in src/app/page.tsx.

Testing

  • Unit Tests Implemented: Tests were created using @testing-library/react for the Help Page. These tests check rendering of FAQ content, search functionality, and accessibility of UI elements. Test files are located in the __tests__ directory.
  • User Tests: Formal user testing was conducted with multiple non-CS testers, including walk-through tasks and exploratory use. Full details and results can be found in the User Test Report. (New in this release)
  • Manual Testing: Manual testing was performed for login/logout, posting announcements, announcement display, and role-based access (admin vs. general users).
  • Selenium IDE Test: An end-to-end test was created using Selenium IDE to simulate admin login, posting an announcement, and logout. The test file is included in the root directory as PostAnnouncementTest.side. You can download the test .side file here.
  • Human-Computer Interaction Tests: Since our webpage is intended for non-technical and elderly users, Human-Computer Interaction (HCI) principles were also considered, including accessibility, color contrast, and navigation clarity using tools such as those listed on the W3C Accessibility Testing Tools site.

Test Coverage (New in this release)

We used Jest with @testing-library/react to measure how much of our Help Page code is covered by automated tests. Firebase was successfully mocked using jest.mock() and a __mocks__/firebaseConfig.ts file.

While we focused testing on the Help Page (our most interactive and static-content-heavy page), other parts of the app rely heavily on dynamic Firebase behavior (e.g., login, announcements), which complicates automated testing.
